# Public Beta Hub

## Purpose

Collect the documents that matter for getting Clawback from internal alpha to a
publicly tryable self-hosted open-source beta.

This folder is for:

- public tryability planning
- public beta operator guidance
- current milestone definition and signoff
- limitations and verification guidance

It is not the full product or architecture archive.

## Read Order

1. `docs/beta/public-tryability-milestone.md`

## What This Folder Should Answer

- what the next milestone actually is
- what "publicly tryable" means for a self-hosted OSS product
- what contract layer must stay fixed across product, provider, and plugin work
- which setup paths are mandatory versus optional
- how much of Gmail, SMTP, Slack, and other providers belongs in product truth
  versus plugin/runtime implementation
- what the current beta supports and what it does not
