## Decision

`0.4` is now signed off as the current public tryable self-hosted OSS beta.

This signoff means Clawback can now honestly claim:

- one supported single-node self-hosted deployment path
- one no-Google first-value path
- one honest reviewed outbound action path
- one real retrieval path with citations
- one coherent verification, limitation, and recovery story

## Gmail Decision

Gmail remains part of the `0.4` milestone as an **optional connected path**.

- a dedicated Gmail acceptance run is **not required** to sign off `0.4`
- the Gmail path remains documented, honest, and optional
- a dedicated Gmail acceptance run is still a useful strengthening step, not a
  beta blocker

What this does **not** mean:

- Gmail is complete enough for launch-level claims
- Gmail setup is zero-friction
- Gmail connected-path evidence is as strong as the no-Google or reviewed-send
  evidence

It means only:

- the path exists
- the path is honestly documented
- the path does not need to block the `0.4` public-beta claim

## Residual Risks Accepted For `0.4`

- Gmail setup remains heavier than the no-Google path
- retrieval quality is keyword-level, not semantic
- deployment remains single-node and self-hosted only
- provider breadth remains intentionally narrow
- some optional connected paths still have weaker evidence than the main
  no-Google and reviewed-send lanes

## Follow-On Work After Signoff

The strongest follow-on packets after `0.4` signoff are:

1. dedicated Gmail connected-path acceptance run
2. better deployment rehearsal and operator tooling
3. stronger retrieval quality evaluation and semantic-search upgrade path
4. broader provider maturity and launch-grade observability

## Conclusion

`0.4` is now accepted as the active public tryable OSS beta milestone.

The remaining work is no longer milestone-definition work. It is follow-on
hardening, stronger optional-path evidence, and launch-oriented expansion.
